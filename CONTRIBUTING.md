# Contributing to AchemDB

Thank you for considering contributing to AchemDB! Your contributions help make this project better for everyone.

## Ground Rules

- Ensure cross-platform compatibility (Linux, macOS, Windows)
- Write tests for new features and bug fixes
- Keep changes focused and atomic - one feature or fix per pull request
- Follow Go code style conventions (run `gofmt` and `golint`)
- Be respectful and welcoming to newcomers

## Getting Started

1. Fork the repository
2. Create a branch for your changes (`git checkout -b feature/your-feature-name` or `fix/your-bug-fix`)
3. Make your changes
4. Ensure tests pass (`go test ./...`)
5. Commit your changes with clear, descriptive messages
6. Push to your fork and submit a pull request

## How to Report a Bug

When filing a bug report, please include:

1. What version of AchemDB are you using?
2. What operating system and architecture are you using?
3. What did you do?
4. What did you expect to see?
5. What did you see instead?

**Security vulnerabilities:** If you find a security vulnerability, you can email the maintainers directly.

## How to Suggest a Feature

Open an issue describing:

- The feature you would like to see
- Why you need it
- How it should work

This helps us understand the use case and discuss the implementation before you start coding.

## Code Review Process

- Pull requests will be reviewed by maintainers
- Feedback will be provided within a reasonable timeframe
- After feedback, please respond within two weeks to keep the PR active
- Once approved, a maintainer will merge your changes

## Commit message convention

The project follow [conventional commit](https://www.conventionalcommits.org/en/v1.0.0/) guideline for commit messages.

## Questions?

Feel free to open an issue for questions or discussions about the project.
